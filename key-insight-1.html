Key Insight 1

Importance of Building a Strong Academic Foundation

Throughout my life, I have always been keen on learning, understanding, and the application of various skills or academic concepts. However, my interest in teaching, and helping people learn was never my day-to-day focus. However, finishing high school and entering UofSC, these priorities quickly shifted. I started being more involved with teaching my fellow classmates’ things such as sports, board games, and even topics in certain classes. I noticed a problem. People often would try to build upon skills that were underdeveloped or rushed, leading to elaborate skill sets being held up by the smallest support. Just as a real foundation’s purpose is to support a larger structure, a weak foundation could result in instability or potential collapse. Similarly, learners often fail to reach true heights with their learning because of the limitations of their weak intellectual foundation in a subject. True mastery of a subject is not defined just by the degree of complexity in one’s topics, but rather the integrity of the intellectual principles they are built upon.

My pursuit of learning in college also led me to new opportunities. Being inspired to pursue research as my primary career path, the idea of becoming a University Professor came to mind. My time as a Teaching Assistant for CSCE146, Algorithmic Design II, allowed me to test this perspective. This class introduced topics such as Big O Notation, Sorting Algorithms, and more. I was thrilled to be in the position to assist students in a class so fundamentally important to their major experience. My primary role throughout my TA position was to be present throughout various lab sessions, answering questions or helping students with concepts they were having difficulty with. As seen in artifact #1, student assignments would follow similar formats throughout labs: Applying a specific data structure or algorithm into an applicable setting. While not focusing on the intricate explanations of how every concept works, the assignments focus on showing real world applications and use cases. One of the best outcomes that came from being a TA was not only watching the students’ skills develop as the class progressed but also recognizing how important these initial concepts were to their career in coding or computing. Furthermore, my concurrent enrollment in CSCE350 (Data Structures and Algorithms) further led me to fulfill these goals.

In CSCE350, previously covered topics such as graphs, data structures, and algorithm efficiency is brought back into the mainstream. These topics are then heavily discussed and practiced, learning more deep insights and the math of why they work. In artifact #2, my final project for the class covered a cohesive analysis pipeline of a famous sorting algorithm Quicksort regarding its efficiency in both time and space. A sorting algorithm is a method for organizing information, like sorting books in a library or sorting people’s heights from shortest to tallest. This algorithm is introduced in classes such as CSCE145 and CSCE146 demonstrating application rather than specific efficiency. However, by already having experience this concept in a more manageable form, the detailed analysis found in the project became significantly easier. Notably, in one of my TA Lab sessions a student asked me a question about one of the algorithms they were learning, “Why do we use this?”. Using the concepts I was learning in CSCE 350 allowed me to explain the underlying mechanics as to the advantages/disadvantages any one algorithm and structure had over another. In domains such as Computer Science, many topics and subjects are clouded by abstraction, making it difficult to understand how and why particular mathematical topics or understandings can be used in computer coding. Being a TA while also taking this class deepened my understanding of both subject matters and emphasized their mutual importance. Although Algorithmic Design II covered relatively simple subject matter, the importance of these introductory topics cannot be overstated. Similarly, without having previously worked with these topics, rigorous explanations of subjects such as graph theory and algorithm analysis can be extremely difficult and tedious. Despite the importance both classes have on the experiences of the students, developing a strong foundation in subject matter in the earlier class pays massive dividends for future, higher level classes. In a research setting, scientists and researchers are breaking ground on topics and problems that very few are familiar with let alone proficient in. By having a strong foundational knowledge, it not only becomes easier to transmit this research information to others, it also leads the way for others to eventually build off of your work.

For a career in research, significant time and energy needs to be dedicated to staying up to date on recent breakthroughs and developments in your field of research. Building upon this can lead to easier engagement with more advanced research topics and more insights. Despite this, however, an equal amount of energy needs to continuously be used for reviewing and strengthening the foundations for which research and development is built upon.

#include <iostream>
#include <vector>
#include <chrono>
#include <fstream>
#include <string>

int partition(std::vector<float>& arr, int left, int right) {
    int middle = left + (right - left) / 2;
    int pivot;

    // Pivot is median of {left, middle, and right} most values of list.
    if ((arr[left] <= arr[middle] && arr[middle] <= arr[right]) ||
        (arr[right] <= arr[middle] && arr[middle] <= arr[left])) {
        pivot = middle;
    } else if ((arr[middle] <= arr[left] && arr[left] <= arr[right]) ||
               (arr[right] <= arr[left] && arr[left] <= arr[middle])) {
        pivot = left;
    } else {
        pivot = right;
    }

    // Swap left-most element with the pivot.
    std::swap(arr[left], arr[pivot]);

    // Since the left-most element is now the pivot, set the pivot value to it.
    float pivotValue = arr[left];

    int i = left + 1;
    int j = right;

    while (true) {
        // While going through list, every value less than pivot, increment i.
        while (i <= right && arr[i] < pivotValue) {
            i++;
        }
        // While iterating through list, every value greater than pivot, decrement j.
        while (j >= left + 1 && arr[j] > pivotValue) {
            j--;
        }

        // i is now greater than j, they have crossed or are equal.
        if (i >= j) {
            break;
        }

        // Swap elements at i and j.
        std::swap(arr[i], arr[j]);
        i++;
        j--;
    }

    // Swap pivot value to the true position.
    std::swap(arr[left], arr[j]);

    // Returns the true position of pivot.
    return j;
}

void quickSort(std::vector<float>& arr, int left, int right) {
    if (left < right) {
        int pivotIndex = partition(arr, left, right);
        // Sorts left side of partition.
        quickSort(arr, left, pivotIndex - 1);
        // Sorts right side of partition.
        quickSort(arr, pivotIndex + 1, right);
    }
}

std::vector<float> readInputFile(const std::string& filename) {
    std::vector<float> arr;
    // Reads input file from name.
    std::ifstream inputFile(filename);

    float element;
    // Adds every element of list to vector.
    while (inputFile >> element) {
        arr.push_back(element);
    }
    inputFile.close();
    return arr;
}

void writeOutputFile(const std::string& filename, const std::vector<float>& arr) {
    // Outputs sorted list in space delimited format.
    std::ofstream outputFile(filename);

    for (size_t i = 0; i < arr.size(); i++) {
        outputFile << arr[i];
        if (i < arr.size() - 1) {
            outputFile << " ";
        }
    }
    outputFile.close();
}

// Appends execution time to executionTime file.
void writeExecutionTime(const std::string& filename, int inputSize, double microseconds) {
    std::ofstream timeFile(filename, std::ios::app);  // Append to file
    timeFile << inputSize << "\t" << microseconds << std::endl;
    timeFile.close();
}

// Averages each set of execution times per size set (ie: 10, 100, 1000)
void calculateAverages() {
    std::ifstream inputFile("Ervin_John_executionTime.txt");

    // Skip header
    std::string header;
    std::getline(inputFile, header);

    double total_10 = 0.0, total_100 = 0.0, total_1000 = 0.0;
    int count_10 = 0, count_100 = 0, count_1000 = 0;

    int inputSize;
    double executionTime;

    while (inputFile >> inputSize >> executionTime) {
        if (inputSize == 10) {
            total_10 += executionTime;
            count_10++;
        } else if (inputSize == 100) {
            total_100 += executionTime;
            count_100++;
        } else if (inputSize == 1000) {
            total_1000 += executionTime;
            count_1000++;
        }
    }
    inputFile.close();

    double avg_10 = total_10 / count_10;
    double avg_100 = total_100 / count_100;
    double avg_1000 = total_1000 / count_1000;

    std::ofstream outputFile("Ervin_John_averageExecutionTime.txt");
    outputFile << "Input Size\tAverage Execution Time" << std::endl;
    outputFile << 10 << "\t" << avg_10 << std::endl;
    outputFile << 100 << "\t" << avg_100 << std::endl;
    outputFile << 1000 << "\t" << avg_1000 << std::endl;
    outputFile.close();
}

int main(int argc, char* argv[]) {
    if (argc == 2 && std::string(argv[1]) == "--average") {
        calculateAverages();
        return 0;
    }
    if (argc == 3) {
        std::string inputFilename = argv[1];
        std::string outputFilename = argv[2];

        // Reads input file into array
        std::vector<float> arr = readInputFile(inputFilename);

        int inputSize = arr.size();

        // Times runtime of quicksort for array.
        auto start = std::chrono::high_resolution_clock::now();
        quickSort(arr, 0, arr.size() - 1);
        auto end = std::chrono::high_resolution_clock::now();
        std::chrono::duration<double> duration = end - start;

        // Outputs sorted array to file.
        writeOutputFile(outputFilename, arr);
        // Appends size and execution time to execution time file.
        writeExecutionTime("Ervin_John_executionTime.txt", inputSize, duration.count());
        return 0;
    }
    return 1;
}
